1.1 System of Linear Equations

A system of linear equations is a collection of equations of the form Ax = b where A is the coefficient matrix, x is the unknown vector, and b is the output vector. In ML, this forms the backbone of linear models.

Forms & Solutions

  • Unique Solution: When rank(A) = rank([A|b]) = n → exactly one solution.
  • Infinite Solutions: rank(A) = rank([A|b]) < n → underdetermined system.
  • No Solution: rank(A) ≠ rank([A|b]) → overdetermined / inconsistent.
Ax = b A⁻¹Ax = A⁻¹b x = A⁻¹b (only when A is invertible)

Key Concepts

  • Vector Dot Product: a · b = Σ aᵢbᵢ — measures similarity/projection.
  • Matrix Multiplication: (AB)ᵢⱼ = Σₖ Aᵢₖ Bₖⱼ — used in weight updates.
  • Transpose: (AB)ᵀ = BᵀAᵀ
  • Eigenvalues/Eigenvectors: Av = λv — used in PCA for dimensionality reduction.
ML Connection: Normal equation for linear regression: θ = (XᵀX)⁻¹Xᵀy solves the system of equations directly without iteration.
1.2 Convex and Non-Convex Functions

Convex Function

A function f is convex if for any two points x₁, x₂ and t ∈ [0,1]:

f(tx₁ + (1-t)x₂) ≤ tf(x₁) + (1-t)f(x₂)
  • Has a single global minimum — gradient descent is guaranteed to find it.
  • Examples: MSE loss, L2 regularization, logistic loss.
  • Second derivative (Hessian) is positive semi-definite: ∇²f ≥ 0.

Non-Convex Function

  • Has multiple local minima and saddle points.
  • Gradient descent may get stuck in local minima.
  • Examples: Neural network loss surfaces, most deep learning objectives.
Why it matters: Convex → guaranteed global minimum. Non-convex → need tricks like momentum, learning rate schedules, multiple restarts.
1.3 Loss Functions and Minimization

A loss function L(ŷ, y) quantifies how wrong the model's prediction ŷ is compared to the true label y. The goal of training is to minimize the average loss over all training samples.

Loss FunctionFormulaUse Case
Mean Squared Error (MSE)L = (1/n)Σ(yᵢ - ŷᵢ)²Regression
Mean Absolute Error (MAE)L = (1/n)Σ|yᵢ - ŷᵢ|Regression (robust to outliers)
Binary Cross-EntropyL = -[y·log(ŷ) + (1-y)·log(1-ŷ)]Binary Classification
Categorical Cross-EntropyL = -Σ yᵢ·log(ŷᵢ)Multi-class Classification
Hinge LossL = max(0, 1 - y·f(x))SVM
Key Insight: MSE penalizes large errors more (quadratic). MAE treats all errors equally. Cross-entropy is derived from maximum likelihood estimation (MLE).
1.4 Gradient and Gradient Descent

What is a Gradient?

The gradient ∇f(θ) is a vector of partial derivatives indicating the direction of steepest ascent. To minimize, we move in the opposite direction.

∂L/∂θⱼ = (1/n) Σᵢ (ŷᵢ - yᵢ) · xᵢⱼ [for MSE linear regression]

Gradient Descent Variants

TypeUpdate RuleProsCons
Batch GDθ = θ − α·∇L(θ) over all dataStable convergenceSlow for large datasets
Stochastic GD (SGD)Update per single sampleFast, online learningHigh variance, noisy
Mini-Batch GDUpdate per batch (32–256 samples)Balance of speed & stabilityNeeds batch size tuning
θ_new = θ_old − α · ∂L/∂θ where α = learning rate (hyperparameter)

Learning Rate Effects

  • Too large α: Overshoots minimum, diverges.
  • Too small α: Very slow convergence.
  • Optimal α: Smooth convergence to minimum.
Exam Tip: Be able to derive the gradient of MSE loss w.r.t. θ manually. ∂MSE/∂θ = (2/n)Xᵀ(Xθ - y)
1.5 Entropy and Information Gain

Shannon Entropy

Entropy H(S) measures the impurity/uncertainty in a dataset S. High entropy = more disorder.

H(S) = -Σ pᵢ · log₂(pᵢ) For binary classification (p = fraction of positive class): H(S) = -p·log₂(p) - (1-p)·log₂(1-p)

Information Gain (IG)

IG measures how much a feature reduces entropy when used to split a dataset. Used in Decision Trees (ID3 algorithm).

IG(S, A) = H(S) - Σᵥ (|Sᵥ|/|S|) · H(Sᵥ) where Sᵥ = subset of S where feature A = v
Example: If a dataset has 5 yes and 5 no: H = -(0.5·log₂0.5 + 0.5·log₂0.5) = 1.0 bit (maximum entropy). Pure dataset H = 0.

Gini Impurity (alternative to entropy)

Gini(S) = 1 - Σ pᵢ²
  • Computationally faster than entropy (no log).
  • Used by CART algorithm (sklearn's DecisionTreeClassifier default).
1.6 Bias-Variance Tradeoff
Total Error = Bias² + Variance + Irreducible Noise Bias² = Error from wrong model assumptions (underfitting) Variance = Error from sensitivity to training data (overfitting)
ScenarioBiasVarianceResult
Simple model (linear)HighLowUnderfitting
Complex model (deep tree)LowHighOverfitting
Optimal modelLowLowGood generalization

Regularization to Control Variance

  • L1 (Lasso): Adds |θ| penalty → sparse model, feature selection. J = L + λΣ|θⱼ|
  • L2 (Ridge): Adds θ² penalty → shrinks weights, no sparsity. J = L + λΣθⱼ²
  • Elastic Net: Combines L1 + L2.
  • λ (lambda): Regularization strength. Higher λ → more regularization → higher bias, lower variance.

2.1 Model Architecture, Selection & Hyperparameter Tuning

Hyperparameter vs Parameter

  • Parameters: Learned from data (weights, biases).
  • Hyperparameters: Set before training (learning rate, depth, C in SVM, k in k-NN).

Tuning Strategies

StrategyHow it WorksBest For
Grid SearchExhaustive search over all combinationsFew hyperparameters, small ranges
Random SearchRandomly samples combinationsMany hyperparameters
Bayesian OptimizationUses surrogate model to guide search intelligentlyExpensive evaluations, large search space

Cross-Validation

K-Fold CV splits data into k folds, trains on k-1, validates on 1. Repeats k times, averages results.

CV Score = (1/k) Σᵢ Score(fold_i) Typical k = 5 or 10
2.2 Linear Regression and Regularization

Simple Linear Regression

ŷ = θ₀ + θ₁x₁ + θ₂x₂ + ... + θₙxₙ = Xθ Minimize: L(θ) = (1/2n) Σ(yᵢ - ŷᵢ)² Normal Equation: θ = (XᵀX)⁻¹Xᵀy

Regularized Variants

Ridge (L2): J(θ) = MSE + λΣθⱼ² Lasso (L1): J(θ) = MSE + λΣ|θⱼ| Elastic Net: J(θ) = MSE + λ₁Σ|θⱼ| + λ₂Σθⱼ²
  • Ridge: All features retained, coefficients shrunk. Good when all features relevant.
  • Lasso: Performs feature selection (coefficients become exactly 0). Good for sparse models.
When to use Normal Equation vs GD: Normal equation is O(n³) — fine for small datasets (<10k features). Use Gradient Descent for large-scale data.
2.3 Classification Algorithms

Logistic Regression

Despite its name, used for binary classification. Applies sigmoid to linear combination:

P(y=1|x) = σ(θᵀx) = 1 / (1 + e^(-θᵀx)) Loss: Binary Cross-Entropy L = -(1/n) Σ [yᵢ·log(ŷᵢ) + (1-yᵢ)·log(1-ŷᵢ)]

k-Nearest Neighbours (k-NN)

  • Non-parametric, lazy learner (no training phase).
  • Classify based on majority vote among k nearest neighbours.
  • Distance metrics: Euclidean, Manhattan, Cosine similarity.
  • Small k → low bias, high variance. Large k → high bias, low variance.

Support Vector Machine (SVM)

Find the hyperplane that maximizes the margin between classes.

Decision boundary: wᵀx + b = 0 Margin = 2 / ||w|| Objective: Minimize ||w||²/2 subject to yᵢ(wᵀxᵢ + b) ≥ 1
  • Hard margin: Linearly separable data only.
  • Soft margin (C parameter): Allows some misclassification. High C → less regularization (overfit).
  • Kernel trick: Maps to higher dimension for non-linear separation. Common kernels: RBF (Gaussian), Polynomial.

Decision Trees

  • Recursively splits data on features that maximize IG (Entropy-based) or minimize Gini.
  • Pruning: Remove branches that add little value → prevents overfitting.
  • Max depth, min samples per split are key hyperparameters.

Random Forest (Ensemble: Bagging)

  • Builds N decision trees on random subsets of data (bootstrap sampling) and random subsets of features.
  • Prediction: Majority vote (classification) or mean (regression).
  • Reduces variance without increasing bias.

Gradient Boosting

  • Builds trees sequentially; each tree corrects errors of the previous.
  • Variants: XGBoost (regularized, parallel), LightGBM (leaf-wise), CatBoost (handles categoricals natively).
  • Loss function is minimized via gradient descent in function space.

Bayesian Methods (Naïve Bayes)

P(y|x) = P(x|y)·P(y) / P(x) [Bayes' Theorem] Naïve assumption: features are conditionally independent given y P(x₁,...,xₙ|y) = Πᵢ P(xᵢ|y)

2.5 Semi-Supervised and Self-Supervised Learning
  • Semi-Supervised: Uses small labeled + large unlabeled data. Pseudo-labeling: train on labeled, predict unlabeled, use high-confidence predictions as labels.
  • Self-Supervised: Creates supervision from the data itself. Contrastive learning (SimCLR): learn that augmented views of same image are similar, different images are dissimilar.
  • Autoencoders: Encoder compresses input → latent space. Decoder reconstructs. Learns compact representations unsupervised.
2.6 Responsible AI in Supervised Learning
  • Fairness Metrics: Demographic parity (equal positive rates across groups), Equalized odds (equal TPR and FPR across groups).
  • Interpretability: SHAP (Shapley values explain each feature's contribution), LIME (local approximation for individual predictions).
  • Data Privacy: Avoid including PII in training data. Check for data leakage (test data leaking into training).
  • Federated Learning: Model trained across decentralized devices without sharing raw data — only model updates are shared.
Data Leakage: When information from outside training set influences the model. Common causes: normalizing before split, using future data, leaky features. Always split first, then preprocess.

3.1 Clustering Algorithms

K-Means Clustering

Partitions n observations into k clusters by minimizing within-cluster sum of squares (WCSS).

Algorithm: 1. Initialize k centroids randomly 2. Assign each point to nearest centroid (Euclidean distance) 3. Recompute centroids as mean of assigned points 4. Repeat 2-3 until convergence WCSS = Σₖ Σᵢ∈Cₖ ||xᵢ - μₖ||²
  • K must be chosen in advance. Use Elbow Method or Silhouette Score.
  • Sensitive to outliers and initialization. Use K-Means++ for better initialization.

Hierarchical Clustering

  • Agglomerative (bottom-up): Start with each point as a cluster, merge closest pairs.
  • Divisive (top-down): Start with one cluster, recursively split.
  • Result visualized as Dendrogram. Cut at height to get k clusters.
  • Linkage: Single (min dist), Complete (max dist), Average, Ward (minimize variance).

DBSCAN (Density-Based)

Parameters: ε (radius), MinPts (min neighbors in radius) Core Point: has ≥ MinPts within ε Border Point: within ε of core but fewer than MinPts neighbors Noise Point: neither core nor border
  • Can find arbitrarily shaped clusters.
  • Automatically identifies outliers as noise.
  • No need to specify k in advance.

Gaussian Mixture Models (GMM)

  • Probabilistic model: data is generated from mixture of k Gaussian distributions.
  • Uses EM (Expectation-Maximization) algorithm to fit.
  • Soft assignments (probability of belonging to each cluster).

3.3 Dimensionality Reduction

Principal Component Analysis (PCA)

Linear technique. Projects data onto directions of maximum variance (principal components).

Steps: 1. Standardize data (zero mean, unit variance) 2. Compute covariance matrix: C = (1/n) XᵀX 3. Compute eigenvectors and eigenvalues of C 4. Sort eigenvectors by eigenvalue (descending) 5. Project data: Z = X · W (W = top-k eigenvectors) Reconstruction Error = ||X - X̂||² (X̂ = reconstructed from k components) Explained Variance Ratio = λᵢ / Σλⱼ
  • Choose k components explaining 95% cumulative variance.
  • Assumes linear relationships. Fails for non-linear data.

t-SNE (t-Distributed Stochastic Neighbour Embedding)

  • Non-linear dimensionality reduction. Primarily for visualization (2D/3D).
  • Preserves local structure: nearby points in high-D stay nearby in low-D.
  • Perplexity hyperparameter controls local vs global structure (typical: 5–50).
  • Cannot be used for new data — must rerun on the entire dataset.
PCA vs t-SNE: PCA is linear, fast, interpretable, works for compression. t-SNE is non-linear, slow, only for visualization, not for downstream ML.

4.1 Perceptron and Basic Terminology
Single Neuron: z = wᵀx + b ŷ = activation(z) Perceptron update rule: w = w + α(y - ŷ)x b = b + α(y - ŷ)

Network Layers

  • Input Layer: Receives raw features (no computation).
  • Hidden Layers: Intermediate transformations. More layers = more capacity.
  • Output Layer: Final prediction (Sigmoid for binary, Softmax for multi-class, Linear for regression).
  • Width: Number of neurons per layer. Depth: Number of layers.

Trainable Parameters

For a layer with nᵢₙ inputs and nₒᵤₜ outputs: Weights: nᵢₙ × nₒᵤₜ Biases: nₒᵤₜ Total: nᵢₙ × nₒᵤₜ + nₒᵤₜ Example: 784→128→64→10 network: Layer 1: 784×128 + 128 = 100,480 params Layer 2: 128×64 + 64 = 8,256 params Layer 3: 64×10 + 10 = 650 params Total: = 109,386 params
4.2 Backpropagation: Chain Rule and Gradient Computation

Chain Rule

Backpropagation applies the chain rule to compute gradients of the loss w.r.t. every weight in the network.

For a 2-layer network: x → z₁=W₁x+b₁ → a₁=σ(z₁) → z₂=W₂a₁+b₂ → ŷ=σ(z₂) → L ∂L/∂W₂ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂W₂ ∂L/∂W₁ = ∂L/∂ŷ · ∂ŷ/∂z₂ · ∂z₂/∂a₁ · ∂a₁/∂z₁ · ∂z₁/∂W₁ δ² = ∂L/∂z₂ (output error signal) δ¹ = (W₂ᵀδ²) ⊙ σ'(z₁) (propagated error) ∂L/∂W² = δ²·a₁ᵀ ∂L/∂W¹ = δ¹·xᵀ
Sigmoid Derivative: σ'(x) = σ(x)·(1-σ(x)) — always between 0 and 0.25. This small value causes the vanishing gradient problem in deep networks.

5.2 Convolutional Neural Networks (CNN)

Convolution Operation

Output feature map: (F * K)[i,j] = Σₘ Σₙ F[i+m, j+n] · K[m,n] Output size: ((W - K + 2P) / S) + 1 W = input width, K = kernel size, P = padding, S = stride Parameters in conv layer: K × K × C_in × C_out + C_out (biases)

Pooling

  • Max Pooling: Takes maximum value in each window. Preserves dominant features.
  • Average Pooling: Takes average. Smoother representation.
  • Reduces spatial dimensions, provides translation invariance.

Typical CNN Architecture

Input → [Conv → BN → ReLU → Pool] × N → Flatten → FC → Output
5.3 Transfer Learning

Use a pre-trained model (trained on large dataset like ImageNet) and adapt it to a new, smaller dataset.

Strategies

  • Feature Extraction: Freeze all pre-trained layers. Only train new classifier head. Use when small dataset and similar domain.
  • Fine-Tuning: Unfreeze some/all layers and continue training with smaller learning rate. Use when larger dataset or different domain.

Popular Pre-trained Models

  • VGG16/VGG19: Simple deep conv nets. Good baseline.
  • ResNet: Residual connections (skip connections) solve vanishing gradient. ResNet50, ResNet152.
  • InceptionV3: Parallel convolutions at different scales.
  • EfficientNet: Scales width, depth, resolution uniformly.
Why Transfer Learning Works: Early CNN layers learn universal features (edges, textures). Later layers learn task-specific features. We reuse the general features and retrain the task-specific ones.
5.4 Sequence Models: RNN, LSTM, GRU

Recurrent Neural Network (RNN)

hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b) ŷₜ = Wᵧhₜ h₀ = zeros (initial hidden state)
  • Processes sequences step by step, maintaining hidden state.
  • Problem: Vanishing/exploding gradient for long sequences.

Long Short-Term Memory (LSTM)

Introduces gating mechanism to control information flow over long sequences.

fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) [Forget Gate — what to discard from cell state] iₜ = σ(Wi·[hₜ₋₁, xₜ] + bi) [Input Gate — what new info to store] C̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) [Candidate cell state] Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ [Cell state update] oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) [Output Gate] hₜ = oₜ⊙tanh(Cₜ) [Hidden state]

GRU (Gated Recurrent Unit)

  • Simplified LSTM with 2 gates (reset and update) instead of 3.
  • Fewer parameters, faster to train. Similar performance to LSTM.

Deep Learning Evaluation Metrics

  • AUC-ROC: Area under Receiver Operating Curve. Threshold-independent.
  • Top-K Accuracy: Is true label in top K predictions? (used in ImageNet).
  • Learning Curves: Training vs validation loss over epochs — diagnose over/underfitting.
  • Convergence Rate: How fast loss decreases per epoch.

RL Loop: State sₜ → Agent → Action aₜ → Environment → Reward rₜ, New State sₜ₊₁ → repeat Goal: Maximize cumulative discounted reward: G = Σₜ γᵗrₜ (γ = discount factor ∈ [0,1])
Discount Factor γ: γ=0 → agent is myopic (only cares about immediate reward). γ=1 → agent values future rewards equally. Typical γ = 0.9 to 0.99.
6.2 Markov Decision Process (MDP)

An MDP is defined by the tuple (S, A, P, R, γ) where:

  • S: Set of states
  • A: Set of actions
  • P(s'|s,a): Transition probability — probability of reaching s' from s by taking action a
  • R(s,a): Reward function
  • γ: Discount factor

Markov Property

The future depends only on the current state, not on the history:

P(sₜ₊₁|sₜ, aₜ, sₜ₋₁, ...) = P(sₜ₊₁|sₜ, aₜ)

Bellman Equation

State Value Function: V(s) = max_a [R(s,a) + γ Σs' P(s'|s,a) V(s')] Action Value Function (Q-function): Q(s,a) = R(s,a) + γ Σs' P(s'|s,a) max_a' Q(s',a') Optimal policy: π*(s) = argmax_a Q(s,a)
6.3 Q-Learning

Model-free RL algorithm. Learns Q(s,a) values directly from experience without knowing the environment model.

Q-Update Rule (derived from Bellman)

Q(s,a) ← Q(s,a) + α[r + γ·max_a' Q(s',a') − Q(s,a)] where: α = learning rate r = reward received s' = next state γ·max Q = discounted future value (TD target) [...] = TD error (Temporal Difference error)

Exploration vs Exploitation (ε-Greedy)

  • Exploitation: Choose action with max Q(s,a) — use current knowledge.
  • Exploration: Choose random action — discover new strategies.
  • ε-Greedy: With probability ε → explore; with (1-ε) → exploit.
  • ε typically decays over time (start high, end low).
Deep Q-Network (DQN): Uses neural network to approximate Q(s,a) for large/continuous state spaces. Used by DeepMind to play Atari games.
